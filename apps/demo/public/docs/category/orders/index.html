Orders

Place, cancel, modify, and inspect orders, and top up the continuation anchor pool that lets a partially-filled order keep working.

📄️Place Order

Submit a hidden, fully-collateralized order — the request body, the cryptographic fields the SDK builds for you, and the response lifecycle.

📄️Cancel Order

Cancel a resting order with a signed cancel request from the owning trading key.

📄️Modify Order

Replace a resting order atomically — an in-place cancel-and-replace with no window where you hold neither order.

📄️Get Order

Read the current status of one order — its state, filled quantity, and the batch it matched in.

📄️Anchor Top-Up

Replenish a partially-filled order's continuation anchor pool so it can keep filling without being re-submitted.
